# 📋 题目描述

题目编号: 49
题目名称：字母异位词分组
难度：中等
标签：哈希表 | 字符串 | 相关企业 | 提示

给你一个字符串数组，请你将字母异位词组合在一起。可以按任意顺序返回结果列表。

字母异位词是通过重新排列不同单词或短语的字母而形成的单词或短语，通常只使用所有原始字母一次。

# 🔍 示例分析

# 示例 1:

输入: strs = ["eat","tea","tan","ate","nat","bat"]
输出: [["bat"],["nat","tan"],["ate","eat","tea"]]

解释:

  • 在 strs 中没有字符串可以通过重新排列来形成 "bat"
  • 字符串 "nat" 和 "tan" 是字母异位词，因为它们可以重新排列形成彼此
  • 字符串 "ate"、"eat" 和 "tea" 是字母异位词，因为它们可以重新排列形成彼此

# 示例 2:

输入: strs = [""]
输出: [[""]]

# 示例 3:

输入: strs = ["a"]
输出: [["a"]]

# 💡 解题思路

# 方法：字符频次编码 + 哈希表

核心思想：字母异位词具有相同的字符频次分布，可以用字符频次作为分组的键值。

算法步骤:

  1. 遍历字符串数组中的每个字符串
  2. 统计每个字符串中各字母的出现频次
  3. 将频次信息编码为唯一的键值字符串
  4. 使用哈希表将具有相同键值的字符串分为一组
  5. 提取哈希表中的所有分组作为最终结果

编码策略:

  • 使用长度为 26 的数组统计 a-z 每个字母的频次
  • 将频次数组编码为字符串格式： #count1#count2#...#count26
  • 相同异位词必然生成相同的编码键值

# 💻 代码实现

# 📊 复杂度分析

  • 时间复杂度: O (n × (m + 26))

    • n 为字符串数组长度，m 为字符串平均长度
    • 每个字符串需要 O (m) 时间统计频次，O (26) 时间生成键值

  • 空间复杂度: O (n × m)

    • 哈希表存储所有字符串，最坏情况下每个字符串都是单独一组

# 💭 解题心得

# ✅ 关键收获：

  • 字符频次编码：将异位词识别问题转化为字符串匹配问题
  • 哈希表分组：利用相同键值自动完成分组操作
  • 编码技巧：使用分隔符避免频次组合产生歧义

# 🔧 实现细节：

  • 使用 vector<int> counts(26, 0) 统计 26 个字母频次
  • 键值编码格式： #1#2#0#... 避免如 "12" 和 "1,2" 的歧义
  • emplace_back() 比 push_back() 效率更高
